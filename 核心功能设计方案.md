### 2.4 草稿自动保存

```javascript
// PublishPost.vue
// 逻辑：
// 1. 表单数据变化后每30秒自动保存到 localStorage
// 2. 页面关闭/路由跳转前保存
// 3. 进入页面时检查草稿
// 4. 弹窗提示用户恢复或忽略

const DRAFT_KEY = 'publish_draft'

// 保存草稿
function saveDraft() {
  const draft = {
    title: form.title,
    content: form.content,
    category: form.category,
    tags: form.tags,
    imageUrls: imageUrls.value,
    timestamp: Date.now()
  }
  localStorage.setItem(DRAFT_KEY, JSON.stringify(draft))
}

// 检查草稿 (onMounted)
function checkDraft() {
  const raw = localStorage.getItem(DRAFT_KEY)
  if (!raw) return null
  const draft = JSON.parse(raw)
  if (Date.now() - draft.timestamp > 24 * 3600 * 1000) {
    localStorage.removeItem(DRAFT_KEY)
    return null  // 超过24小时清除
  }
  return draft
}

// 恢复草稿
function restoreDraft(draft) {
  form.title = draft.title
  form.content = draft.content
  form.category = draft.category
  form.tags = draft.tags
  imageUrls.value = draft.imageUrls
  localStorage.removeItem(DRAFT_KEY)  // 恢复后清除
}

// 发布成功后
function clearDraft() {
  localStorage.removeItem(DRAFT_KEY)
}
```

## 四、地图服务集成方案

### 4.1 为什么集成地图

徒步论坛天然适合集成地图：
- 路线攻略帖子需要展示徒步路线轨迹
- 活动帖子需要标注集合地点
- 用户可以在个人主页标记徒步足迹

### 4.2 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 地图引擎 | Leaflet.js | 3KB gzip，MIT开源，无API Key |
| 底图瓦片 | OpenStreetMap | 免费商用，无需付费 |
| 安装 | `npm install leaflet` | 前端依赖，无需后端改动 |

### 4.3 实现场景

**场景A：帖子详情页显示路线地图**

```
帖子分类 = "路线攻略" 且 extrainfo 中包含 coordinates 时 →
帖子正文下方显示 Leaflet 地图，绘制路线轨迹线
```

```vue
<template>
  <div v-if="routeCoordinates && routeCoordinates.length > 0" class="mt-8">
    <h3 class="text-lg font-bold mb-4">📍 路线轨迹</h3>
    <div ref="mapContainer" class="w-full h-[400px] rounded-2xl overflow-hidden border"></div>
  </div>
</template>

<script setup>
import { ref, onMounted, nextTick } from 'vue'
import L from 'leaflet'
import 'leaflet/dist/leaflet.css'

const mapContainer = ref(null)

onMounted(async () => {
  await nextTick()
  if (!mapContainer.value) return

  const map = L.map(mapContainer.value).setView([30.5, 104.0], 10)
  L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    attribution: '© OpenStreetMap'
  }).addTo(map)

  // 如果有点标记
  if (props.lat && props.lng) {
    L.marker([props.lat, props.lng]).addTo(map)
  }

  // 如果有路线坐标
  if (props.coordinates?.length) {
    const polyline = L.polyline(props.coordinates, {
      color: '#16a34a',
      weight: 4
    }).addTo(map)
    map.fitBounds(polyline.getBounds(), { padding: [30, 30] })
  }
})
</script>
```

## 六、设计原则

1. **向后兼容**：所有新字段通过 ALTER TABLE 添加，不破坏现有数据
2. **增量推进**：每个阶段功能独立可上线，无阻塞依赖
3. **零成本**：地图免费(Leaflet+OSM)、草稿免费(localStorage)、BBCode 自研
4. **纯文本优先**：内容存储为纯文本 + BBCode，渲染时解析为 HTML，不影响全文检索
5. **低耦合**：地图组件通过 props 接收数据，不依赖全局状态
